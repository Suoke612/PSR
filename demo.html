核心设计理念

专为单人开发者 + AI 协作设计的轻量化工作流

📄

文档即状态机

每个开发步骤对应一个 Markdown 文件。文件存在 = 步骤完成，文件缺失 = 需要执行。主智能体检查目录就知道该干什么，永远不会迷路。

🤖

专用子智能体协作

研究、审查、修复由三个专门的子智能体分别负责。各司其职，互不越界——研究员只看不改，审查员只评不改，修理工只修不评。

🔄

对话中断也能恢复

所有状态记录在磁盘文件中。即使对话丢失、时隔数月回来，系统自动从文档重建上下文，不需要重新解释项目。

🎯

一个 Phase 一个对话

每个功能模块独立为一个 Phase，在一个对话中完成讨论→研究→计划→执行→审查→测试全流程，上下文不爆炸。

📦

最小执行单元

每个 Plan 产出不超过 3 个文件，确保每次改动范围可控，审查和回滚都轻松。

🔁

自动状态恢复

新对话开始时，系统自动扫描文档时间戳，判断当前 Phase 进度，无需手动同步。

🛡️

三道质量闸门

每个 Phase 必经代码审查 → 修复阻塞项 → 用户验收测试，架构违规不会累积。

📋

检查表驱动

主智能体通过检查表自动判断该执行哪一步——不需要记住复杂的流程步骤。

⏸️

延后修复不阻塞

发现 bug 但依赖未实现系统？记录到 SUMMARY.md，后续 Phase 自动继承，当前流程不卡死。

三个专用子智能体

各司其职，质量把关——所有审查标准来自 CLAUDE.md，换项目只需改一个文件

🔍

PSR-Researcher

研究员

扫描代码库、分析影响范围、检查已知陷阱。支持项目级、里程碑级、Phase 级三级研究。

🔒 只读代码 · 输出 RESEARCH.md

📋

PSR-Reviewer

审查员

对照架构约束审查代码，输出 🔴🟡🟢 三级分级报告。支持常规审查和全局框架审查两种模式。

🔒 只读代码 · 输出 REVIEW.md

🔧

PSR-Fixer

修理工

读取 🔴 阻塞项，逐个修复并验证。最多 3 轮迭代，确保架构违规清零。

🔓 可修改代码 · 追加执行记录

为什么选择 PSR？

专为单人开发者 + AI 协作打造的轻量化方案

🆚 相比裸用 AI 编程

  • 状态不丢：文档存在磁盘上，对话丢失也能恢复。不需要每次开场白解释项目。
  • 质量不退化：每个 Phase 强制审查和修复，架构违规不会累积。不是"写完就忘"。
  • 上下文不炸：3 层架构分离关注点，CLAUDE.md 保持精简。一个对话只处理一个 Phase。
  • 不破坏已有系统：修改前强制确认影响范围，Researcher 扫描后才动手。

🆚 相比传统项目管理

  • 零配置：只是一堆 Markdown 文件，不需要安装任何工具或插件。
  • AI 原生：子智能体自动读取 CLAUDE.md 了解项目规则，换项目无需重配。
  • 单人优化：没有多人协作的 ceremony，每一步就是下一件事。
  • 无学习成本：不需要学新工具，会写 Markdown 就能用。
🪶

极致轻量

没有数据库、没有服务端、没有配置界面。只有 Markdown 文件和目录结构。

👤

单人优先

不需要 stand-up、不需要 code review 会议。你和 AI 两个人就能跑完整个流程。

💬

📁

文件即进度

看一眼目录就知道项目做到哪了。不需要打开任何看板或仪表盘。

🔌

随时接入

🌙
